getCachedMessageList() → {Array.<Message>}

Get the cached message list Retrieves the current message list cached in the client, does not go to the ZypHub server. The message list may be out of sync with the server if connectivity has been interrupted. The user must be logged in otherwise an error will be emitted
Returns:
returns the cached message list, or null if there is an error

newMessage(msg) → {boolean}

Send a new message
Parameters:
Name Type Description
msg Object The message to be sent
Properties
Name Type Attributes Description
pattern string The message pattern. One of FCFS, STANDARD, FYI
members Array Array of identifiers (ZypHub or Enterprise) to send the message to
content string The body of the message
maxAccepts number <optional>
 If the pattern if FCFS this sets the number of 'Accepts' to consider the message complete, if the pattern is FCFS and this parameter is not specified it will default to 1
Fires:
  • event:MessageNotification
Returns:
returns if the request was successfully made, a successful request may still result in an error being emitted. A successful message will emit the messages id on the 'new' topic
Type
boolean
